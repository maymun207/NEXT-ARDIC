# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it
responsibly. **Do not open a public GitHub Issue.**

### Contact

- **Email:** [security@yourdomain.com](mailto:security@yourdomain.com)
- **Expected response time:** 48 hours

### What to Include

- Description of the vulnerability
- Steps to reproduce the issue
- Affected component (e.g., API route, frontend, dependency)
- Potential impact assessment
- Suggested fix (if available)

### What to Expect

1. We will acknowledge your report within **48 hours**
2. We will investigate and provide a timeline for resolution
3. We will notify you when the issue is fixed
4. We will credit you (if desired) in the fix commit

## Scope

This policy covers:

- This repository (`nextjs-enterprise-starter` or your project name)
- All API routes and server-side logic
- Client-side code and dependencies
- Infrastructure configuration (Vercel, Supabase)

## Disclosure Policy

- This repository may be private or public — update this accordingly
- Please do not disclose vulnerabilities publicly until fixed
- We follow responsible disclosure practices

## Dependency Security

- **Dependabot** monitors dependencies for known vulnerabilities
- **npm audit** runs in CI on every push and pull request
- **Weekly audit reports** are generated automatically (Monday 09:00 UTC)
- Patch-level updates are auto-merged; minor/major require review
